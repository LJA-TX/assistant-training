# Publication Readiness Checklist

## Purpose

Standardize pre-push publication hygiene review for milestone checkpoints.

## Trigger

Use when determining whether the repository is ready for publication/push approval.

## Checklist

- [ ] Confirm required closure/readiness artifacts for the target milestone are present.
- [ ] Confirm repository status is clean or differences are explicitly documented as intentional.
- [ ] Confirm no unintended tracked files are present.
- [ ] Confirm no temporary artifacts are staged.
- [ ] Confirm no ZIP bundle leakage into tracked paths.
- [ ] Confirm fixture/catalog reconciliation artifacts are present for the targeted milestone.
- [ ] Confirm no obvious documentation inconsistencies within the targeted publication scope.
- [ ] Produce determination:
  - ready to push, or
  - not ready to push (with issue list and corrective actions)

## Boundary Notes

- Publication readiness does not execute push.
- Push occurs only after explicit authorization.
