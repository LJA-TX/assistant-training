# Phase ZE Recommendation

## Recommended Execution Path

Run the topology sweep in low-to-high dispersion order:

1. Control: compact H1/H2-like local footprint
2. Treatment A: two-cluster footprint
3. Treatment B: four-cluster footprint
4. Treatment C: fully dispersed footprint

## Why This Path Has The Highest Information Value

This is the smallest controlled experiment that tests the only remaining variable identified by Phase ZD.

It is preferable to another broad corpus redesign because it keeps every known confounder fixed:

- anchor concentration,
- cue retention,
- safety block,
- tool-positive density,
- frozen scaffold,
- and validation surfaces.

The only moving part is the topology of the positive patch.

## Why The Control Should Be Compact

H1 and H2 were successful with a compact patch-local footprint:

- `100` patch rows,
- roughly `20%` corpus-span locality,
- frozen non-tool slices,
- exact-tool-request cue on the positive rows.

That is the natural baseline for topology perturbation.

## Why Not Another Full Redesign

A broad redesign would reintroduce the same confounders already seen in:

- Phase L,
- Phase Q,
- Phase U,
- and the Z-series anchor sweep.

Those phases already showed that moving multiple levers at once does not identify the remaining causal variable.

## Success Signal

The experiment is informative if the capability metrics move systematically with dispersion.

The strongest result would be:

- compact topology best,
- more dispersed topologies worse,
- with safety remaining within bounds.

That would support patch-local topology as causal.
