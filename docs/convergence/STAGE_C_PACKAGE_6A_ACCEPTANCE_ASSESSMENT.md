## Basis

### Determinations 1-2 Basis

The evidence set is broader than a single branch recommendation artifact.

It includes:

1. a success-path retrospective from Package `3C`;
2. an evidence-availability constrained branch from Packages `4A`-`4C`;
3. a blocker-oriented branch from Packages `5A`-`5E`;
4. repeated full-run evidence for the blocker-oriented branch.

That is enough to support a governance recommendation about regimen structure.

### Determination 3 Basis

The blocker-oriented branch is reusable because it now has:

1. defined entry logic;
2. repeated-run stability evidence;
3. blocker characterization discipline;
4. attribution discipline;
5. an explicit blocker decision point.

This is more than an isolated exception.

### Determination 4 Basis

Repository evidence still shows a meaningful shared foundation:

1. authoritative fact emission;
2. row identity;
3. passive governance consumption;
4. passive reconciliation;
5. readiness and gate framing.

So the best representation is not branch-first.

It is a common lifecycle trunk with conditional branches.

### Determination 5 Basis

`conditionally adopt` is strongest because it balances two realities:

1. the branch has enough evidence to merit formal recognition;
2. blocker-oriented evidence still comes from one active surface family and has not yet traversed a downstream post-blocker transition.

### Determination 6 Basis

Package 6A is recommendation-only.

It does not:

1. adopt the branch;
2. change doctrine;
3. change detector or threshold behavior;
4. reopen any readiness, gate, or planning state.
