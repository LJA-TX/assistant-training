## 1. Branch Retrospective

### Original Branch Objectives

The blocker-oriented branch began to answer three connected questions:

1. whether `direct_answer_substitution_count` was a legitimate active Stage C surface on the frozen corpus;
2. whether its blocked state reflected a stable governed blocker rather than absent evidence or transient runtime drift;
3. whether the blocker could be characterized, bounded, and taken through a legitimate post-blocker transition path without violating preservation rules.

### Branch Entry Conditions

At entry:

1. reconciliation state was `requires_future_migration`
2. readiness state was `migration-blocked`
3. gate state was `gate-blocked`
4. the frozen corpus already contained the governed tool-positive denominator
5. authoritative row identity, governance consumption, reconciliation, and readiness framing were already in place

### Major Milestones

Major blocker-branch milestones:

1. `5A`: established legitimate lifecycle entry
2. `5B`: established strong blocker persistence across repeated full runs
3. `5C`: characterized the blocker population
4. `5D`: separated governance-preserving missingness from pathway incompleteness
5. `5E`: extracted blocker-oriented regimen lessons
6. `6A-6B`: conditionally adopted the blocker-oriented branch into the reusable regimen
7. `7A`: showed no second blocker-oriented surface family currently exists
8. `7B-7D`: established, authorized, and executed the first real post-blocker transition investigation
9. `7E-7J`: carried the branch through bounded planning, design, readiness, and execution authorization framing
10. technical spike: tested the smallest governance-safe scorer-pathway change in runtime
11. runtime forensics: localized the dominant remaining uncertainty to actual emitted runtime outputs
12. legacy-surface validity assessment: established that the legacy surface is materially misaligned with its claimed semantics
13. `7K`: determined that the branch should transform into a different investigation family

### Major Findings

Major findings accumulated by the branch:

1. the blocker is real
2. the blocker is stable
3. the blocker population splits into `131` structurally incapable rows and `3` ambiguous rows
4. the smallest safe scorer-pathway change is runtime-inert on the frozen corpus
5. the authoritative path is not hiding a meaningful clean direct-answer or scalar population
6. the legacy `direct_answer_substitution_count` surface is materially misaligned with its name
7. the dominant remaining uncertainty is runtime-output and corpus behavior

### Final Outcomes

Final branch outcomes:

1. the blocker-oriented branch is now a valid reusable regimen component
2. the direct-answer surface remains operationally unchanged:
   - reconciliation `requires_future_migration`
   - readiness `migration-blocked`
   - gate `gate-blocked`
3. the original blocker question is substantially answered for the current surface and frozen corpus
4. the branch has reached a clean stopping point
5. the next high-value work belongs to a different investigation family
